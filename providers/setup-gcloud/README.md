# setup-gcloud

Tinx provider for `gcloud`, modeled on `google-github-actions/setup-gcloud` while following the Tinx provider/runtime/workspace split.

## Behavior mapping

- Input: version pinning is accepted through `GCLOUD_VERSION`, `CLOUDSDK_VERSION`, or `INPUT_VERSION`.
- Output equivalent: the installer prints `binary_path`, and the workspace runtime resolves `gcloud` on `PATH` after materialization.
- PATH mutation: handled by Tinx workspace shims and the local tool runtime, not by the provider binary itself.
- Download strategy: resolves `latest` from the Google Cloud SDK components manifest, downloads a versioned CLI archive from the public Cloud Storage release bucket, validates the archive against the published object MD5 from the same bucket, and materializes the full SDK tree under the tool install directory.
- Platform coverage: `linux`, `darwin`, and `windows` with `amd64` and `arm64`.

## Design

- Bundled installer tool: `setup-gcloud`
- User-facing tool: `gcloud`
- Workspace install path: `.workspace/tools/.../bin/gcloud` via a thin launcher generated by the provider
- Global cache: `~/.tinx/cache/providers/setup-gcloud`
- Version resolution:
  - `latest` or `stable` resolves through `components-2.json`
  - `565.0.0` or `v565.0.0` installs that exact version

## Build And Package

```bash
go test ./...
tinx release \
  --manifest provider.yaml \
  --main ./cmd/tinx-provider-gcloud \
  --dist dist \
  --output oci \
  --tag v0.1.0
```

## Example workspace flow

```bash
cat > demo/tinx.yaml <<'EOF'
apiVersion: tinx.io/v1
kind: Workspace
workspace: demo
providers:
  gcloud:
    source: ghcr.io/sourceplane/tinx-setup-gcloud:v0.1.0
EOF

tinx init demo/tinx.yaml
GCLOUD_VERSION=565.0.0 tinx -w demo exec -- gcloud version
```

## Local validation

```bash
workspace_dir=$(mktemp -d)
cat > "$workspace_dir/tinx.yaml" <<EOF
apiVersion: tinx.io/v1
kind: Workspace
workspace: setup-gcloud-test
providers:
  gcloud:
    source: $(pwd)/oci
EOF

tinx init "$workspace_dir/tinx.yaml"
GCLOUD_VERSION=565.0.0 tinx -w "$workspace_dir" exec -- gcloud version
```

## Publishing

```bash
cd providers/setup-gcloud
tinx release \
  --manifest provider.yaml \
  --main ./cmd/tinx-provider-gcloud \
  --dist dist \
  --output oci \
  --tag v0.1.0 \
  --push ghcr.io/<org>/tinx-setup-gcloud:v0.1.0
```
