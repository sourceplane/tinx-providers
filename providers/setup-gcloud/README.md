# setup-gcloud

kiox provider for `gcloud`, modeled on `google-github-actions/setup-gcloud` while following the current kiox provider package, runtime, and workspace model.

## Behavior mapping

- Input: version pinning is accepted through `GCLOUD_VERSION`, `CLOUDSDK_VERSION`, or `INPUT_VERSION`.
- Output equivalent: the installer prints `binary_path`, and kiox resolves `gcloud` on `PATH` after lazy materialization.
- PATH mutation: handled by kiox workspace shims and the local tool runtime, not by the provider binary itself.
- Download strategy: resolves `latest` from the Google Cloud SDK components manifest, downloads a versioned CLI archive from the public Cloud Storage release bucket, validates the archive against the published object MD5 from the same bucket, and materializes the full SDK tree under the tool install directory.
- Platform coverage: `linux`, `darwin`, and `windows` with `amd64` and `arm64`.

## Design

- Bundled installer tool: `setup-gcloud`
- User-facing tool: `gcloud`
- Managed install path: `$KIOX_HOME/store/<storeID>/tools/gcloud/bin/gcloud` via a thin launcher generated by the provider
- Global cache: `~/.kiox/cache/providers/setup-gcloud`
- Version resolution:
  - `latest` or `stable` resolves through `components-2.json`
  - `565.0.0` or `v565.0.0` installs that exact version

## Build And Package

```bash
go test ./...
kiox release \
  --manifest provider.yaml \
  --main ./cmd/kiox-provider-gcloud \
  --dist dist \
  --output oci \
  --tag v0.2.0
```

## Example workspace flow

```bash
mkdir -p demo
cat > demo/kiox.yaml <<'EOF'
apiVersion: kiox.io/v1
kind: Workspace
metadata:
  name: demo
providers:
  gcloud:
    source: ghcr.io/sourceplane/setup-gcloud:v0.2.0
EOF

kiox init demo
kiox --workspace demo ls
kiox --workspace demo status
GCLOUD_VERSION=565.0.0 kiox --workspace demo -- gcloud version --format=json
```

## Local validation

workspace_dir=$(mktemp -d)

apiVersion: kiox.io/v1
kind: Workspace

GCLOUD_VERSION=565.0.0 kiox --workspace "$workspace_dir" -- gcloud version --format=json

## Publishing

```bash
cd providers/setup-gcloud
kiox release \
  --manifest provider.yaml \
  --main ./cmd/kiox-provider-gcloud \
  --dist dist \
  --output oci \
  --tag v0.2.0 \
  --push ghcr.io/<org>/setup-gcloud:v0.2.0
```
